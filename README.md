# Yakovenkona Bot

Telegram-бот для ознакомления с тарифами услуг дизайнера интерьера и расчета стоимости дизайн-проекта.  
Бот присылает описание тарифов со ссылкой на пример, принимает площадь помещения, рассчитывает стоимость по тарифу, сохраняет историю действий и позволяет связаться с дизайнером.

## Возможности

- Расчет стоимости по площади.
- Поддержка нескольких тарифов.
- Сценарии с кнопками и состояниями.
- Сохранение истории действий пользователей.
- Админский просмотр базы через `admin_db.py`.
- FAQ-раздел с изображениями.

## Структура проекта

- `assets/faq/` — изображения для раздела FAQ.
- `database/` — код работы с базой данных.
- `handlers/` — обработчики команд, callback-кнопок и сценариев.
- `keyboards/` — inline и reply-клавиатуры.
- `states/` — состояния бота.
- `utils/` — вспомогательные модули.
- `loader.py` — создание бота и подключение фильтров.
- `main.py` — точка входа приложения.
- `admin_db.py` — консольный просмотр данных базы.
- `requirements.txt` — зависимости проекта.

## Требования

- Python 3.14+.
- Telegram Bot Token.
- Установленные зависимости из `requirements.txt`.

## Установка

1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/USERNAME/REPO.git
   ```

2. Перейдите в папку проекта:
   ```bash
   cd REPO
   ```

3. Создайте виртуальное окружение:
   ```bash
   python -m venv .venv
   ```

4. Активируйте окружение:
   ```bash
   .venv\Scripts\activate
   ```

5. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

6. Создайте файл `.env` по образцу `.env.template` и заполните токен бота.

## Запуск

Запустите бота командой:

```bash
python main.py
```

Для просмотра базы данных из консоли:

```bash
python admin_db.py
```

## Конфигурация

Перед запуском нужно указать:
- `BOT_TOKEN`
- другие параметры, если они используются в `config_data`

## Работа с базой

В проекте используется SQLite на старте разработки.  
Файл базы создается автоматически и хранится локально как `bot.db`.

База сохраняет:
- пользователей;
- события;
- историю пользовательских сообщений;
- результаты расчета.

Административный просмотр базы доступен через `admin_db.py`.
Команды:
python admin_db.py count-users (покажет количество пользователей)
python admin_db.py count-events (покажет количество записей в истории)
python admin_db.py list-users (выведет последних пользователей)
python admin_db.py list-events (выведет последние события)
python admin_db.py history 123456789 (покажет историю конкретного пользователя)
